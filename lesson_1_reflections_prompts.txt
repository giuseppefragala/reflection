A - Pros: I can decide when commit, saving all the changing I need.
    Cons: In case of making too much changes, it can be hard finding difference from some version to another.

Q - Why do you think some version control systems, like Git, allow saving multiple files in one commit, while others, like Google Docs, treat each file separately?
A - The most useful property of Git is to compare together all file related to a commit, after their changes.

Q - How can you use the commands git log and git diff to view the history of files?
A - With git log command you can see when a commit was made and it is possible to have an idea of that was changed, by reading the comment. Using git diff it is possible find the changes made from a version to the other.
